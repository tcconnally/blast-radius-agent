Project Details

What it does
Blast Radius Analyzer is a GitLab Duo custom agent that helps developers understand the downstream impact of code changes before they make them. When a developer mentions @blast-radius with a file path in a GitLab issue or MR comment, the agent queries GitLab Orbit's knowledge graph to locate the file's definition, traverses all gl_reference edges to find every file/function/module that depends on it, recursively traces transitive dependencies, identifies cross-project impact, classifies risk as Low/Medium/High/Critical, and posts a structured risk report as an issue comment.
How I built it
Platform: GitLab Duo Agent Platform — custom agent hosting, invocation via issue/MR mentions, and tool enablement for posting comments. Knowledge Graph: GitLab Orbit — the structured, queryable codebase representation. The agent uses get_graph_schema to discover node types and query_graph to traverse the dependency graph using Cypher-style queries. The LIKE wildcard escaping uses exact-match (=) + LIKE with ESCAPE clause for safe suffix matching. Agent Skill: A reusable skill file (skills/blast-radius/SKILL.md) following the Agent Skills specification. Includes precise query patterns for direct dependencies, transitive traversal, cross-project impact, and risk classification rules. Local Engine: A zero-dependency Python engine (blast_radius/) wraps the Orbit CLI with cycle-safe traversal, exclude filtering, LIKE literal escaping, and deterministic risk scoring. 29 tests pass, CI green on Python 3.9/3.11/3.12. Demo video: Terminal simulation built with Playwright — 5 scenes, auto-scrolling, color-coded roles and badges, rendered at 1280x720 and converted to H.264.
Why GitLab Orbit
GitLab Orbit is the critical enabler that makes blast radius analysis possible at scale. Before Orbit, dependency analysis required grep (misses transitive chains), IDE find-references (no cross-repo awareness), or manual review (doesn't scale). Orbit's graph structure (gl_definition nodes connected by gl_reference edges) maps perfectly to the dependency traversal problem. A reverse graph walk from any definition reveals everything that depends on it. The agent uses Orbit meaningfully — recursive graph traversal with depth control, cycle detection, and cross-project scope.
What's next
Visual dependency graph rendering (Mermaid/Graphviz in issue comments), CI gate integration that blocks merge if risk is Critical without review, IDE integration as a VS Code extension showing dependency impact as you type, historical hotspot analysis to identify files becoming dangerous hubs over time, and scheduled weekly audits of high-churn directories to surface risky patterns before they become incidents.
